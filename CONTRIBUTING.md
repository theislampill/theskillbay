# Contributing

## Branch/Patch Expectations
- Feature branches
- Small, focused commits
- Git history preserved

## Test Requirements
- All new code tested
- Existing tests pass
- Coverage for crypto, policy, flows

## Documentation Updates
- Update docs for architecture changes
- Add examples for new features

## Minimal Patch
- Solves one problem
- Includes tests
- Updates docs if needed

## Architecture Changes
- Must update ARCHITECTURE.md
- Discuss in issues first

## Stubs
- Clearly marked as STUB
- Not half-implemented

## Deferred
- Not silently added
- Marked in ROADMAP.md
